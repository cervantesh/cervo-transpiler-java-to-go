# Support

Use GitHub issues for support that can be handled publicly and reproducibly.

## Good Support Requests

- Minimal Java input or fixture.
- Exact `j2go` command.
- Operating system and Go version.
- Expected output.
- Actual output.
- Any `JTG` diagnostics or parser errors.

## Not Appropriate For Public Support

- Customer source code.
- Private repository URLs.
- Access tokens, API keys, logs with secrets, or credentials.
- Confidential migration reports.
- Legal approval questions about third-party code ownership.

For private security or confidentiality issues, follow `SECURITY.md`.
