# Modern Java To Go Transpiler Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Cervo Java-to-Go transpiler as a modern compiler pipeline: Java source -> ANTLR parse tree -> semantic analysis -> Cervo IR -> Go generator -> `gofmt`/golden tests.

**Architecture:** The current Flex/Bison/C++ implementation remains as a legacy reference while the modern implementation is built in Go under `cmd/` and `internal/`. ANTLR4 generates a Java parser, the lowerer converts parse trees into an explicit IR, semantic passes validate names/types/features, and the Go emitter uses `go/ast` plus `go/format` instead of printing target code from grammar rules.

**Tech Stack:** Go 1.25, ANTLR4 tool jar, `github.com/antlr4-go/antlr/v4 v4.13.1`, Java grammar from a controlled ANTLR subset first, `go/ast`, `go/token`, `go/format`, golden tests, GitHub issues for delivery tracking.

## Tool Decision

| Layer | First implementation | Why |
| --- | --- | --- |
| Parser | ANTLR4 generated Go parser | Modern equivalent to Flex/Bison with visitors/listeners and cleaner separation. |
| Java grammar | Reduced Java subset grammar in `grammar/JavaSubset.g4` | Keeps first MVP testable while preserving a path to `antlr/grammars-v4`. |
| Semantic analysis | Go package `internal/semantic` | Keeps Java-specific rules out of parser and emitter. |
| IR | Go package `internal/ir` | Stable compiler boundary between Java frontend and Go backend. |
| Go output | `go/ast` and `go/format` | Produces valid formatted Go without ad hoc string output as the core representation. |
| Tests | `go test` golden tests | Verifies exact generated Go and rejects unsupported Java deterministically. |
| AI | Optional sidecar after deterministic reports | AI helps explain and suggest; it does not own default code generation. |

## MVP Scope

Supported Java in the first modern MVP:

- one or more simple `public class` declarations
- `public static void main(String[] args)`
- static methods
- primitive types: `int`, `double`, `boolean`, `String`, `void`
- local variable declarations
- assignments: `=`, `+=`, `-=`
- expressions: literals, identifiers, calls, unary `!`/`-`, binary arithmetic, comparisons, `&&`, `||`
- `if` / `else`
- `while`
- simple `for`
- `return`
- `System.out.println(...)`
- basic arrays as a diagnostic boundary first, not full lowering

Explicitly rejected with structured diagnostics:

- inheritance
- interfaces
- generics
- overloads
- exceptions
- packages/imports beyond report metadata
- reflection
- streams/lambdas
- threads/synchronization
- class fields and instance methods until the struct/receiver model is designed

- [ ] **Step 1: Create grammar**

Create `grammar/JavaSubset.g4` with a controlled Java subset that supports classes, static methods, local variables, control flow, calls, and expressions. The grammar must include a single `compilationUnit` start rule and generate visitors.

Required start shape:

```antlr
grammar JavaSubset;

compilationUnit
    : packageDecl? importDecl* typeDecl* EOF
    ;

packageDecl
    : 'package' qualifiedName ';'
    ;

importDecl
    : 'import' qualifiedName ('.' '*')? ';'
    ;

typeDecl
    : classDecl
    | ';'
    ;

classDecl
    : modifier* 'class' Identifier classBody
    ;
```

The full first grammar must include:

- `methodDecl`
- `formalParameters`
- `typeType`
- `block`
- `statement`
- `localVariableDecl`
- `expression`
- `primary`
- literals
- comments and whitespace skipping

```go
package parser

import (
	"github.com/antlr4-go/antlr/v4"

	"github.com/cervantesh/cervo-transpiler-java-to-go/internal/parser/gen"
)

func ParseSource(fileName string, source string) (gen.ICompilationUnitContext, []Diagnostic) {
	input := antlr.NewInputStream(source)
	lexer := gen.NewJavaSubsetLexer(input)
	listener := newErrorListener(fileName)
	lexer.RemoveErrorListeners()
	lexer.AddErrorListener(listener)

	tokens := antlr.NewCommonTokenStream(lexer, antlr.TokenDefaultChannel)
	p := gen.NewJavaSubsetParser(tokens)
	p.RemoveErrorListeners()
	p.AddErrorListener(listener)
	p.BuildParseTrees = true

	return p.CompilationUnit(), listener.Diagnostics()
}
```

`newErrorListener` must implement ANTLR error-listener callbacks and store `Diagnostic` values.

```go
type Kind int

const (
	KindInvalid Kind = iota

	KindBoolean
	KindInt
	KindDouble
	KindString
	KindArray
	KindObject
)

type Type struct {
	Kind Kind
	Name string
	Elem *Type
}

func (t Type) GoName() string {
	switch t.Kind {
	case KindVoid:
		return ""
	case KindBoolean:
		return "bool"
	case KindInt:
		return "int"
	case KindDouble:
		return "float64"
	case KindString:
		return "string"
	case KindArray:
		if t.Elem == nil {
			return "[]any"
		}
		return "[]" + t.Elem.GoName()
	case KindObject:
		if t.Name != "" {
			return t.Name
		}
		return "any"
	default:
		return "any"
	}
}
```

```go
package pipeline

import (
	"fmt"

	"github.com/cervantesh/cervo-transpiler-java-to-go/internal/emitgo"
	"github.com/cervantesh/cervo-transpiler-java-to-go/internal/lower"
	"github.com/cervantesh/cervo-transpiler-java-to-go/internal/parser"
)

type Result struct {
	Code        []byte
	Diagnostics []error
}

func TranspileSource(fileName string, source string) Result {
	tree, syntax := parser.ParseSource(fileName, source)
	if len(syntax) != 0 {
		out := make([]error, 0, len(syntax))
		for _, diagnostic := range syntax {
			out = append(out, diagnostic)
		}
		return Result{Diagnostics: out}
	}

	irFile, semantic := lower.Lower(fileName, tree)
	if len(semantic) != 0 {
		out := make([]error, 0, len(semantic))
		for _, diagnostic := range semantic {
			out = append(out, diagnostic)
		}
		return Result{Diagnostics: out}
	}

	code, err := emitgo.Emit(irFile)
	if err != nil {
		return Result{Diagnostics: []error{fmt.Errorf("emit Go: %w", err)}}
	}
	return Result{Code: code}
}
```
